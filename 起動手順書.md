# 筋トレ履歴ダッシュボード 起動手順書

## 1. 前提

- Node.js がインストールされていること
- 作業フォルダ: `/Users/kaamisyama/Documents/筋トレ/workout-review-app`
- このアプリは追加ライブラリを使っていないため、通常は `npm install` は不要です

Node.js が入っているか確認する場合:

```sh
node -v
```

## 2. 起動方法

ターミナルでアプリのフォルダへ移動します。

```sh
cd /Users/kaamisyama/Documents/筋トレ/workout-review-app
```

開発サーバーを起動します。

```sh
npm run dev
```

起動に成功すると、ターミナルに次のような表示が出ます。

```text
Workout review app running at http://0.0.0.0:4173/
Local access: http://127.0.0.1:4173/
```

ブラウザで次のURLを開きます。

```text
http://127.0.0.1:4173/
```

## 3. 使い方

- 初期状態では `data/sample.csv` が読み込まれます
- 自分のCSVを見る場合は、画面右上の「CSVを選択」からCSVファイルを選びます
- 期間フィルターで「全期間」「直近30日」「直近60日」「直近90日」を切り替えられます
- 部位タブから種目を選ぶと、種目ごとの推移グラフを確認できます

## 4. 終了方法

起動しているターミナルで `Ctrl + C` を押します。

## 5. ポートを変更したい場合

標準では `4173` 番ポートで起動します。
別のポートで起動したい場合は、次のように指定します。

```sh
PORT=3000 npm run dev
```

この場合は、ブラウザで次のURLを開きます。

```text
http://127.0.0.1:3000/
```

## 6. うまく起動しないとき

### `npm: command not found` と表示される

Node.js が入っていない可能性があります。Node.js をインストールしてください。

### `EADDRINUSE` と表示される

同じポートを別のアプリが使っています。別のポートを指定して起動してください。

```sh
PORT=3000 npm run dev
```

### 画面が表示されない

次を確認してください。

- ターミナルで `npm run dev` が起動したままになっている
- ブラウザで `http://127.0.0.1:4173/` を開いている
- 別ポートで起動した場合は、そのポート番号のURLを開いている
